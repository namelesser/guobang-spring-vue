# 项目优化计划

## 概览

四阶段修复已完成，以下是按优先级分类的进一步优化机会。分为 **安全加固**、**前端重构**、**后端优化**、**DevOps 完善**、**代码清理** 五大类。

---

## 一、安全加固（高优先级）

### 1.1 Docker 安全
- `backend/Dockerfile`: 添加非 root 用户运行容器
- `docker-compose.yml`: 移除硬编码默认密码（`${PG_PASSWORD:-552300}`、`${TRANSPORT_AUTH_SECRET:-guobang-secret-key-2024}`）
- `docker-compose.yml`: 生产环境不暴露数据库端口 5432
- `docker-compose.yml`: 添加资源限制（CPU/内存）

### 1.2 nginx 安全头
- `frontend/nginx.conf`: 添加 `X-Frame-Options`、`X-Content-Type-Options`、`X-XSS-Protection`、`Referrer-Policy`
- `frontend/nginx.conf`: 添加 `server_tokens off`
- `frontend/nginx.conf`: 添加 `client_max_body_size 20M`
- `frontend/nginx.conf`: 配置代理超时

### 1.3 环境变量
- `.env.prod`: `VITE_SERVICE_BASE_URL` 指向 Mock 服务器（`https://mock.apifox.cn/...`），需改为实际后端地址
- `.gitignore`: 添加 `.env.*` 忽略规则（除 `.env.example`）

## 二、前端重构（中高优先级）

### 2.1 提取 `useCollections()` composable（~200 行）
**问题**: `loadCollections()`、`collectionCache`、`opts()` 在 4 个视图中完全重复。
**涉及文件**:
- `records/index.vue:180-192`
- `review/index.vue:95-111`
- `rates/index.vue:157-169`
- `report/index.vue:59-71`

**方案**: 创建 `frontend/src/hooks/business/use-collections.ts`，返回 `collectionCache` 和 `optionsFor(category)` 方法。

### 2.2 提取 `useDebouncedSearch()` composable（~100 行）
**问题**: `searchTimer`、`abortController`、`scheduleSearch()`、`buildParams()` 在 records 和 images 视图中重复。
**涉及文件**:
- `records/index.vue:24-25, 212-232`
- `images/index.vue:21-22, 176-195`

**方案**: 创建 `frontend/src/hooks/business/use-debounced-search.ts`。

### 2.3 提取共享工具函数
**问题**: `today()`、`monthOptions`、`fmtNum`、`firstImageId`、导出下载逻辑在多个视图重复。
**方案**: 创建 `frontend/src/utils/business.ts`。

### 2.4 消除 `any` 类型（~52 处）
**重点**: `DataQualityResponse` 接口有 20 个 `any[]` 字段 + 冗余嵌套 `report?` 属性。
**方案**: 为每个检查结果定义具体接口（`FutureDateRow`、`MissingWeightRow` 等），删除冗余 `report?`。视图中的 `rows: any` 改为已有的 `TransportRecord[]`、`ImageAsset[]` 等。

### 2.5 删除未使用代码（~371 行）
- `frontend/src/hooks/business/useAbortController.ts`（29 行）— 从未导入
- `frontend/src/hooks/business/auth.ts`（21 行）— 从未导入
- `frontend/src/hooks/common/table.ts`（321 行）— Soybean Admin 脚手架，从未使用
- `fetchImageThumbnail`、`fetchDataQualityDetail` — 从未调用
- `CollectionsResponse` 接口 — 从未导入

### 2.6 性能优化
- `home/index.vue`: 合并双 `<script>` 块，列定义提取为 `computed` 或模块常量
- `data-quality/index.vue:75-190`: `sections` computed 每次创建新列数组，应提升到模块级
- `review/index.vue:89-93`: `getOptions()` 每次调用创建新数组

### 2.7 更新项目元信息
- `frontend/package.json`: `name` 从 `soybean-admin` 改为 `guobang-transport`
- `frontend/.env`: `VITE_APP_TITLE` 从 `SoybeanAdmin` 改为 `国邦运输管理系统`

## 三、后端优化（中优先级）

### 3.1 RecordService 未使用 Lombok
- `RecordService.java` 仍使用手动构造函数注入（第 44-48 行），应改为 `@RequiredArgsConstructor`

### 3.2 RecordService 缺少 `@RequiredArgsConstructor`
- 与项目其他 Service 不一致（ImageService、DataQualityService 等均已用 Lombok）

### 3.3 WebConfig 转发逻辑已过时
- `WebConfig.java` 转发 SPA 路由到 `index.html`，但 static 目录已删除，这些转发会 404
- 应删除或修改为只在有 static 资源时启用

### 3.4 Actuator 健康检查
- `application.yml` 缺少 `management.endpoints` 配置
- Docker 健康检查无法使用 Actuator

### 3.5 连接池优化
- `application.yml:15`: 最大连接数 30 偏高，建议 10-15
- 缺少 `connection-timeout`、`idle-timeout`、`max-lifetime`

### 3.6 DatabaseInitializer 与 Flyway 重复
- `DatabaseInitializer.java` 的 SQL 与 `V1__init_schema.sql` 完全重复
- 考虑移除或仅在 Flyway 禁用时执行

```bash

cd frontend && pnpm typecheck && pnpm lint && pnpm fmt

# 后端
mvn compile && mvn test
```

```bash

TRANSPORT_AUTH_PASSWORD=552300 TRANSPORT_AUTH_SECRET=guobang-secret-key-2024 PG_PASSWORD=552300 ./mvnw spring-boot:run
```
